# Workshop content must be timeless

Never reference real-world, time-bound, or engagement-specific
details in any artifact or output — no calendar dates, times,
timezones, headcounts, locations, or scheduling logistics. The
workshop material must read the same regardless of when or to
whom it is delivered.

**Why:** the repository is the durable, reusable workshop
material; baking in one engagement's specifics dates it and ties
it to a single audience.

**How to apply:** express structure in timeless terms (e.g.
"three 2-hour sessions", "Session 1/2/3") rather than named days
or dates. Describe the audience by role, skill level, and
industry, not by count, location, or organization. Applies
retroactively — scrub such details from files that already
contain them. Pairs with [[feedback_no_company_names]].
