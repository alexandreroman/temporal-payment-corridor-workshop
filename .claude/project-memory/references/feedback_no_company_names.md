# Never reference company names

Never write any company or organization name in any artifact or
output — source code, comments, docs, commit messages, memory
notes, the workshop spec/materials, or conversational prose. This
covers the workshop partner, their clients, and any other named
company. Refer to them generically instead (e.g. "the partner",
"a systems-integrator partner", "major banks", "the LLM
provider").

**Why:** these materials are shareable/version-controlled, and
the engagement is confidential; a named company would expose a
client relationship.

**How to apply:** when company-specific context is needed, phrase
it by role, industry, or category rather than by name. Applies
retroactively — scrub any company name from files that already
contain one.
