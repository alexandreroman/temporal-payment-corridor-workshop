# Memory entries are present-tense standing facts

Every project-memory entry states what is currently true, in
the present tense. It never narrates past events or
anecdotes — no "renamed from X", "the user asked to…", "we
changed…", "previously it was…". Only the durable current
fact belongs in memory.

**Why:** memory is read by future sessions as standing truth;
historical narration is noise that ages badly and misleads
once the event no longer matters.

**How to apply:** before saving, rewrite any historical
phrasing as the present fact — e.g. "the package is named
`codec`", not "`codec_server` was renamed to `codec`". The
rule covers the body, the `name`, and the `description`
fields alike.
